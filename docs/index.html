Erisia NoteBlocks.json

Arraste pastas com arquivos .ogg ou selecione pelo navegador de arquivos.

Arraste uma pasta de áudios

ou clique para escolher arquivos

Suporta múltiplas pastas contendo arquivos .ogg.

Pré-visualização do arquivo sounds.json

Como funciona

  1. Arraste uma pasta com arquivos .ogg para a área indicada ou clique em “Selecionar pastas”.
  2. O sistema captura toda a estrutura de subpastas e cria chaves seguindo o caminho pasta.subpasta.arquivo.
  3. Arquivos numerados com sufixos _1, _2 etc. são agrupados em um mesmo registro, gerando uma lista de variações para reprodução aleatória.
  4. Após o processamento, revise o sounds.json e baixe o arquivo para incluir no recurso do servidor.

Exemplo de entrada

Sounds/
 └─ littleroom/
    └─ boreal_druid/
       ├─ druid_vox_1.ogg
       ├─ druid_vox_2.ogg
       └─ howl.ogg

Resultado esperado

{
  "littleroom.boreal_druid.druid_vox": {
    "sounds": [
      "littleroom/boreal_druid/druid_vox_1",
      "littleroom/boreal_druid/druid_vox_2"
    ]
  },
  "littleroom.boreal_druid.howl": {
    "sounds": [
      "littleroom/boreal_druid/howl"
    ]
  }
}

Erisia ModelData Mixer

Prepare overrides de custom_model_data mesclando arquivos existentes com novos modelos.

Novos modelos (.json)

Arraste arquivos com overrides a serem adicionados ou selecione pelo navegador.

Aceita múltiplos arquivos ou pastas.

Modelos atuais (minecraft/models/item)

Arraste a pasta existente ou selecione pelo navegador.

Utilize a pasta oficial do resource pack.

Pré-visualização dos arquivos

Como funciona

  1. Carregue os arquivos .json com novos overrides no painel “Novos modelos”.
  2. Carregue a pasta atual minecraft/models/item no painel “Modelos atuais”.
  3. Clique em “Mesclar arquivos” para gerar os novos valores de custom_model_data.
  4. Revise os arquivos modificados e baixe o pacote .zip para substituir no resource pack.

Exemplo de resultado

{
  "parent": "item/generated",
  "textures": {
    "layer0": "item/diamond_sword"
  },
  "overrides": [
    { "predicate": { "custom_model_data": 1001 }, "model": "item/diamond_sword/base" },
    { "predicate": { "custom_model_data": 1002 }, "model": "item/diamond_sword/flame" },
    { "predicate": { "custom_model_data": 1003 }, "model": "item/diamond_sword/frost" }
  ]
}

Conversor Nexo/Oraxen → ItemsAdder

Converta automaticamente seus itens do Nexo ou Oraxen para o formato ItemsAdder.

Arraste a pasta do plugin

ou clique para escolher a pasta

Suporta pastas do Nexo ou Oraxen (deve conter pasta items e opcionalmente pack).

Conversão concluída

Como funciona

  1. Arraste a pasta completa do plugin Nexo ou Oraxen (deve conter a pasta items).
  2. O sistema detecta automaticamente se é Nexo ou Oraxen e converte os arquivos YAML.
  3. Os resource packs (pasta pack) são copiados e adaptados para ItemsAdder.
  4. Baixe o arquivo ZIP pronto para ser colocado na pasta ItemsAdder/contents/ do servidor.

Estrutura esperada

PluginFolder/
 ├─ items/
 │  ├─ item1.yml
 │  └─ item2.yml
 └─ pack/ (opcional)
    └─ assets/
       └─ ...

Resultado

ItemsAdder/
 └─ contents/
    └─ oraxen/ (ou nexo)
       ├─ item1.yml
       ├─ item2.yml
       └─ resource_pack/
          └─ assets/
             └─ ...
